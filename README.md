# MuseScore 移動ド自動入力ツール

MuseScoreで作成された楽譜データ（`.mscz` ファイル）を読み込み、選択した楽器パートの歌詞（第2行目）へ自動的に「移動ド」を書き込む、ブラウザ完結型のWebアプリケーションです。

## 主な機能

* **移動ド算出**: 楽譜の調号（キー）と音符の実音（ピッチ）から、正確な階名を自動計算して付与します。移調楽器（サックスなど）のパートでも、実音ベースで正しく計算されます。
* **4大移動ド方式とカスタムマッピング**:
  * 短調（マイナー）の主音設定（ドレミ式 / ラシド式）および派生音の表記（記号式 / 変名式）を柔軟に選択できます。
  * **カスタムマッピング機能**: 鍵盤UIを用いて、短調時の各音程（異名同音のフォールバック含む全17枠）に対する独自の階名を定義できます。設定したカスタムルールは JSON ファイルとして書き出し・読み込みが可能です。
* **範囲指定機能**: ルビを振る範囲を指定できます（複数指定可）。未入力の場合は全体にルビを振ります。
* **転調区間の自動スキャンと個別・一括設定**: 楽譜内の調号変化（転調）を自動で検知し、区間ごとに「長調」か「短調」かを設定できます。転調が多い楽曲向けに「すべて長調/短調」の一括設定ボタンも備えています。
* **記載ロジック**:
  * **同音の省略**: 同じ高さの音が連続する場合はルビを省略するか選択できます。ただし、小節の頭の音符にはルビを振ります。
  * **タイ・スラーの考慮**: タイやスラーで繋がっている音符には全角アンダーバー（＿）を記載します（小節の頭の音符であっても）。タイ明けの音符には強制的にルビを振ります。スラーでつながっている音符は括弧書きでルビを振ります。
  * **ピボット（橋渡し）表記**: 転調直後の最初の1音目は `(旧階名→)新階名` の形式で出力します。
* **和音のスキップ**: ピアノなどの伴奏パートにおける複数音（和音）の重なりは自動的に検知してスキップし、単旋律にのみルビを振ります。
* **既存歌詞への追記処理**: 対象パートの歌詞2行目にすでに別の入力（アーティキュレーションのメモなど）が存在する場合、元のデータを上書きせず、後方に半角スペースを空けて移動ドを追記します。
* **対応バージョン**: 内部のXML構造（音符、ピッチ、歌詞タグ）の編集に依存しているため、MuseScore 3 および MuseScore 4 の両方のファイル形式で動作します。
* **ブラウザ完結**: すべての処理はローカルのブラウザ内で完結し、外部サーバーへのデータ送信は行いません。
* **上書きの防止**: 変換後のファイルは、元のファイル名に `_solfege` という接尾辞が付与されてダウンロードされます。元の `.mscz` ファイルが直接上書きされることはありません。

## MuseScore 4 での利用に関する補足

本ツールは MuseScore 3 と MuseScore 4 の両方に対応していますが、MuseScore 4 ではファイルの内部構造にいくつかの変更があるため、以下の点にご注意ください。

### eid 属性について

MuseScore 4.2 以降で保存されたファイルでは、各XML要素に `eid`（Element ID）という一意の識別属性が付与されるようになりました。本ツール（v2）では、ファイル読み込み時に `eid` の有無と形式（数値型 / 英数字型）を自動検出し、新規作成する `<Lyrics>` 要素にも適切な `eid` を付与します。また、ファイル先頭の `<LastEID>` 値も自動的に更新します。

* MuseScore 4.4 以前では `eid` は数値型（例: `eid="12345"`）です。
* MuseScore 4.5 以降では `eid` は英数字型（例: `eid="4a7b3c9d1e2f"`）に変更されています。
* MuseScore 3 のファイルには `eid` 属性がないため、`eid` は付与されません（MuseScore 3 での動作に影響はありません）。

### ZIP 再圧縮時の互換性について

`.mscz` ファイルの実態は ZIP アーカイブであり、本ツールでは JSZip ライブラリを使用して展開・再圧縮を行っています。MuseScore 4 では ZIP 内部のファイル構成が拡張されており（`audiosettings.json`、`viewsettings.json`、`score_style.mss` 等が追加）、本ツールはこれらのファイルを変更せずそのまま保持します。

ただし、まれに再圧縮後のファイルを MuseScore 4 で開いた際に「ファイルが破損しています」等の警告が表示されることがあります。これは ZIP の圧縮方式やエントリ順序の差異に起因するもので、多くの場合は「無視して開く」を選択すれば問題なく利用できます。**万が一の事態に備え、元の `.mscz` ファイルは必ずバックアップを取ってからご利用ください。**

### 前方互換性について

MuseScore のネイティブファイル形式は下位互換（新しいバージョンで古いファイルを開く）はサポートされていますが、前方互換（古いバージョンで新しいファイルを開く）はサポートされていません。本ツールで出力したファイルは**入力ファイルと同じバージョン形式のまま出力される**ため、MuseScore 4 で保存されたファイルを処理した結果を MuseScore 3 で開くことはできません。これは本ツールの制限ではなく、MuseScore 自体の仕様です。

MuseScore 4 形式のファイルを MuseScore 3 で開きたい場合は、別途公開している **[MuseScore 4 → 3 変換ツール](https://midokaoru.github.io/mscz-converter/)** をご利用ください。バージョンヘッダーやタグ名の差異を自動変換し、MuseScore 3 で読み込める形式に変換します。こちらもブラウザ完結型で、楽譜データが外部に送信されることはありません。なお、変換結果の完全性を保証するものではありませんので、元ファイルのバックアップを取ったうえでご利用ください。

## 注意事項

* **レイアウトが崩れるリスク**:
  * 本ツールは歌詞（第2行目）へ記載する為、譜面のレイアウトが縦に伸びます。その為、ツール適用前には三行おさまっていた譜面が二行までしかおさまらない、などのレイアウト崩れが発生する可能性があります。
  * 強弱記号などを五線譜の下側に配置するとルビと重なってしまう可能性があるので、五線譜の上に配置しておくことを推奨します。
* **リズムの浄書について**: 本ツールにはリズムを浄書する機能は備わっていません。ミューズスコアでリズムの浄書を行ってから本ツールをご利用ください。
* **バージョン間の非互換に注意**: MuseScore 4 で作成・保存したファイルを本ツールで処理した後、MuseScore 3 で開くことはできません（詳しくは上記「前方互換性について」を参照）。
* **元ファイルのバックアップ**: 本ツールは元のファイルを上書きしない設計ですが、念のため作業前に元の `.mscz` ファイルのバックアップをお勧めします。


## 内部の処理フロー

`.mscz` ファイルは、実態としてはXMLファイル（楽譜データ本体）などを圧縮したZIPファイルです。本ツールでは `JSZip` ライブラリを使用し、以下の手順でファイルを直接書き換えています。

1. **ファイルの展開と事前スキャン**
   アップロードされた `.mscz` をメモリ上で解凍し、楽譜データ本体（`.mscx`）のパスを特定します。XML内の `<Measure>` と `<KeySig>` タグをスキャンし、転調が行われている小節リストと、対象となる楽器パートリストを抽出してUIに表示します。MuseScore 4.2 以降のファイルでは `<LastEID>` タグを検出し、eid 属性の形式を自動判定します。

2. **移動ドの計算とタグの書き換え**
   ユーザーが指定した調性設定および表記ルール（またはカスタムマッピング辞書）に基づき、各音符に対して階名を計算します。
   * MuseScore内部の `tpc` (Tonal Pitch Class) 値と `pitch` (MIDIノート番号) を参照し、異名同音を正確に判別します。
   * 音符の連続、タイ（`<Tie>`）、小節線またぎなどの条件を判定し、書き込みが必要な `<Note>` タグにのみ `<Lyrics>` タグを追加（または既存ノードにテキストを追記）します。和音（`<Chord>`内の2つ目以降の`<Note>`）はスキップします。
   * MuseScore 4.2 以降のファイルでは、新規作成する `<Lyrics>` 要素に一意の `eid` 属性を付与し、処理完了後に `<LastEID>` の値を更新します。

3. **再圧縮と出力**
   移動ドの歌詞データが追記された `.mscx` データを再圧縮し、新しい `.mscz` ファイルとしてブラウザ上からダウンロードさせます。MuseScore 4 で追加された付随ファイル（`audiosettings.json` 等）は変更せずそのまま保持されます。
