# MuseScore 移動ド自動入力ツール

MuseScoreで作成された楽譜データ（`.mscz` ファイル）を読み込み、選択した楽器パートの歌詞（第2行目）へ自動的に「移動ド」を書き込む、ブラウザ完結型のWebアプリケーションです。

## 主な機能

* **移動ド算出**: 楽譜の調号（キー）と音符の実音（ピッチ）から、正確な階名を自動計算して付与します。移調楽器（サックスなど）のパートでも、実音ベースで正しく計算されます。
* **各移動ド方式に対応**: 短調（マイナー）の主音を何と呼ぶか（ドレミ式 / ラシド式）、派生音をどう表記するか（ファ#などの記号式 / フィなどの変名式）を柔軟に選択できます。
* **転調区間の自動スキャンと個別設定**: 楽譜内の調号変化（転調）を自動で検知し、区間ごとに「長調（メジャー）」か「短調（マイナー）」かを設定できます。一括設定ボタンも備えています。
* **浄書・省略ロジック**:
  * **同音の省略**: 同じ高さの音が連続する場合はルビを省略し、楽譜が文字で埋め尽くされるのを防ぎます。
  * **小節・タイの考慮**: タイで繋がっている音符は省略し、タイ明けの音符や、小節の頭の音符には強制的にルビを振ります。
  * **ピボット（橋渡し）表記**: 転調直後の最初の1音目は `(旧階名→)新階名` の形式で出力し、スムーズな読譜をサポートします。
* **和音のスキップ**: ピアノなどの伴奏パートにおける複数音（和音）の重なりは自動的に検知してスキップし、単旋律にのみルビを振ります。
* **既存の歌詞への影響の排除**: 本ツールは `<Lyrics>` タグの第2行目（`<no>1</no>`）に対して移動ドを書き込みます。すでに2行目に別の歌詞やテキストが存在する場合、元のテキストの後ろに半角スペースを挟んで移動ドが記載されます。
* **ブラウザ完結**: すべての処理はローカルのブラウザ内で完結し、外部サーバーへのデータ送信は行いません。

## 注意事項

* **リズムの浄書について**: MuseScore上で手入力・MIDIインポートされた複雑なリズム（拍の頭をまたぐ音符など）がある場合、本ツールを使用する前にMuseScore本体の機能である「ツール」＞「リズムの再グループ化」を実行しておくことを強く推奨します。これにより、タイの判定などがより正確になります。
* **上書きの防止**: 変換後のファイルは、元のファイル名に `_solfege` という接尾辞が付与されてダウンロードされるため、元のデータを誤って上書きすることはありません。

## 内部の処理フロー

`.mscz` ファイルは、実態としてはXMLファイル（楽譜データ本体）などを圧縮したZIPファイルです。本ツールでは `JSZip` ライブラリを使用し、以下の手順でファイルを直接書き換えています。

1. **ファイルの解凍と事前スキャン**
   アップロードされた `.mscz` (ZIP形式) をメモリ上で解凍し、楽譜データ本体（`.mscx` ファイル）のパスを特定します。その後、XML内の `<Measure>` と `<KeySig>` タグをスキャンし、転調が行われている小節のリストと、対象となる楽器パート（`<Part>` / `<Staff>`）のリストを抽出してUIに表示します。

2. **移動ドの計算とタグの書き換え**
   ユーザーが指定した調性設定と表記ルールに基づき、各音符に対して計算を行います。
   * MuseScore内部の `tpc` (Tonal Pitch Class) 値と `pitch` (MIDIノート番号) を参照し、異名同音（例：ファ#とソ♭）を楽譜上の見た目通りに正確に判別します。
   * 音符の連続、タイ（`<Tie>`）、小節線またぎなどの条件を判定し、書き込みが必要な `<Note>` タグにのみ、移動ドのテキストを持った `<Lyrics>` タグを追加します。

3. **再圧縮と出力**
   移動ドの歌詞データが追記された `.mscx` データを再圧縮し、新しい `.mscz` ファイルとして出力・ダウンロードさせます。

## 使用ライブラリ
